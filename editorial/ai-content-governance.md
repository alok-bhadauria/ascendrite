# AI Content Governance

## Document Metadata
*   **Purpose**: Outlines criteria for validating AI-generated content, human review thresholds, and prompt guidelines.
*   **Scope**: Governs all AI-generated curriculum files, prompt directories, and moderation queues.
*   **Intended Audience**: AI engineers, content editors, and QA reviewers.
*   **Related Documents**:
    *   [AI Philosophy](../docs/governance/ai-philosophy.md)
    *   [AI Architecture](../docs/architecture/ai-architecture.md)
*   **Ownership**: Head of Editorial Division & Quality Assurance Lead

---

## 1. Principles of AI Generation

While AI agents are utilized to draft, format, and translate educational assets, they must operate under strict system constraints to prevent quality degradation:

*   **No Direct Publication**: AI agents cannot write directly to public branches. All AI-generated inputs must be submitted as PR proposals and routed through review queues.
*   **Syllabus Integrity**: AI agents are restricted to generating content that aligns with the parameters defined in `curriculum-map.json` and `domain-taxonomy.json`.
*   **Technical Authority**: Explanations, math, and code must be cross-referenced with authoritative documentation, official compiler specifications, or verified libraries.

---

## 2. Human-in-the-Loop Review

Every pull request initiated by an AI agent must be routed through the moderation queue:
*   **Review Priority**: High-level concepts, proofs, and code examples require manual verification and sign-off by a designated subject moderator.
*   **Rejection Triggers**: AI-generated PRs must be rejected immediately if they contain syntax errors, missing citations, mathematical inaccuracies, or generic explanations.

---

## 3. Hallucination Prevention

To prevent model hallucinations in technical topics:
*   **RAG-Enforced Prompts**: Prompt templates must enforce RAG retrieval steps, injecting schema references and topic contexts before generating content.
*   **Unit Tests**: Code examples generated by AI must execute successfully within sandbox containers, verifying that inputs return target outputs before PR submission.
*   **Fact Validation**: Mathematical equations and proofs must be verified against academic references.

---

## 4. Citation and Attribution

*   **Explicit Citations**: Technical statements (such as algorithm complexity limits or RFC behaviors) must link to the canonical source (e.g., standard libraries, specifications, or academic publications).
*   **Attribute Metadata**: AI-authored assets must include metadata tagging that identifies the model, date of generation, prompt template used, and verifying human editor.
