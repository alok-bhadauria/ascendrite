## Acronym Definitions

*   **ADR (Architectural Decision Record)**: A document capturing a technical decision, its context, alternatives considered, and consequences.
*   **DTO (Data Transfer Object)**: An object encapsulating data sent across network layers or domain interfaces, decoupling execution layers from data schemas.
*   **FSM (Finite State Machine)**: A computation model consisting of states, inputs, and transitions, governing entity lifecycles.
*   **HLD (High-Level Design)**: An architectural overview outlining system components, database choices, and system-wide data flows.
*   **JWT (JSON Web Token)**: An open standard (RFC 7519) defining a compact, self-contained way for securely transmitting information between parties as a JSON object.
*   **PITR (Point-in-Time Recovery)**: A database backup configuration enabling recovery of data state at any specified timestamp.
*   **RAG (Retrieval-Augmented Generation)**: An AI search strategy that retrieves authoritative data from structured repositories before prompt execution.
*   **RBAC (Role-Based Access Control)**: An authorization system restricting platform actions based on defined actor roles.
*   **RTO (Recovery Time Objective)**: The target duration of time within which a system must be restored after a service interruption.
*   **RPO (Recovery Point Objective)**: The maximum acceptable period of data loss during an operational failure.
*   **SLA (Service Level Agreement)**: A formal commitment defining target platform availability and operational response speeds.
*   **SLO (Service Level Objective)**: Target metrics (such as latency thresholds) that operational systems must satisfy.
*   **SSOT (Single Source of Truth)**: The practice of structuring data and configurations so that every element has a unique, authoritative master definition.
